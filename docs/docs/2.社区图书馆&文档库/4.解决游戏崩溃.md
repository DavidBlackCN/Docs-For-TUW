> 在==纪念版==的更新中，进行了一些现代优化和调整，修复了许多历史遗留问题，以下仅列举**《亡灵战争·弥赛亚》**中仍可能遇到的崩溃问题。
>
> 如果你正在游玩老版本地图，可以考虑查看 [(旧)游戏崩溃](/docs/(旧)游戏崩溃/) ！

## ==MC崩溃了，回到了主菜单？==

- 在**【纪念版】**中进行了一项优化，当MC出现问题并崩溃时，并不会回到启动器界面，而是回到主菜单，方便再次进入游戏！
  
- 在MC崩溃时，你可以找到MC的**错误报告(crash-report)**或**游戏日志(logs)**并进行分析，解决问题后再进入游戏！

## ==错误报告/日志在哪里？==

- 在**【纪念版】**的启动器中默认开启了版本隔离，所以游戏根目录路径应该是：`.minecraft\versions\版本名\`，示例文件树：

::: file-tree
- .minecraft #游戏根目录
  - versions
    - 【纪念版】亡灵战争·弥赛亚 #版本名
      - config #配置文件夹
        - forge.cfg
        - mod-1.toml
        - mod-2.txt
        - mod-3.properties
      - mods #模组文件夹
        - mod-1.jar 
        - mod-2.jar 
        - mod-3.jar 
      - resourcepacks #资源包文件夹
        - resources.zip
      - shaderpacks #光影文件夹
        - shader.zip
      - saves #存档文件夹
        - [存档]亡灵战争·弥赛亚
      - crash-reports #错误报告
        - ++ crash-2025-03-21_09.27.46-server.txt
      - logs #游戏日志
        - 2026-04-27-1.log.gz
        - 2026-04-27-2.log.gz 
        - ++ lastest.log #游戏最后一次运行的日志
:::

如果游戏崩溃生成了`crash-report`则优先使用`crash-report`  
如没有生成`crash-report`则可以使用`lastest.log`

## ==我该如何求助他人？==
1. **截图报告**（而不是手机拍屏）并向群友求助
2. 如果报告/日志较长，请考虑使用**日志分享网站**！
::: info 推荐
- **[国内]** [Logshare.CN](https://logshare.cn/)
- **[海外]** [Pastebin.com](https://pastebin.com/)

参考教程：[Bilibili/[教程]Logshare——您不可错过的日志分享网站](https://www.bilibili.com/opus/1196242735198109701)
:::

## ==崩溃造成了卡关？==

有时候在剧情进行时的崩溃可能会导致游戏卡关，可使用 [常用坐标收集](/docs/常用坐标收集/) 内的坐标传送至下一个场景。

- **MDS私改剧情专用线路：可选章节继续剧情**
```command
/tp @a -888 38 41
```

- **6.5章最后的防守战：闪退后再进入游戏剧情将强制停止，不要继续等待，输入指令继续剧情**
```command
/setblock -1772 66 697 152
```
